After a pause of some months I got back to work.

Meanwhile I learned a lot about AI agents and how to use them. They are a
powerful tool.

First I applied the rules for AI agent so it knows how to handle code in
general, but also some rules for Rust and GDScript.

With the help of AI I did restructure the repository: from a **multi-repo** to a
**mono-repo**. Also the protobuf and server were moved from `server` directory
to the root directory. Now the files structure is much cleaner.

I thought if how to manage the project and I reached to the conclusion that a
need a milestones system. So I created a new board on Trello for this. I created
3 milestones based on how many people I want to test the game:

1. After **Milestone #1** I want to be able to test the game with some close
   friends and have 100 questions in-game;
2. After **Milestone #2** I want to be able to test the game with more friends
   and friends of the friends and have 200 questions in-game;
3. After **Milestone #3** I want to be able to launch the game to the public and
   have 300-400 questions in-game.

So I created in Trello 6 lists: the current in-progress tasks, a list for each
milestone and a list for pre-milestone tasks.

Because it took some time to understand again how the game should have worked
(talking about the matches) and what was the main hook of the game I realised
that I needed a better way to document how the game works beside the document I
did in Excalidraw. So I asked AI for help and it suggested to create FSM
diagrams (Finite State Machine diagrams) and this is what I did.

Here you can see the FSM for the server-side for a match:

```mermaid
stateDiagram-v2
    [*] --> LobbyFoundPlayers
    LobbyFoundPlayers --> DraftQuestions
    DraftQuestions --> CalculatePingOfAllPlayersPhase
    CalculatePingOfAllPlayersPhase --> WaitAllPlayersReadyPhase : ping calculated for all players
    WaitAllPlayersReadyPhase --> WaitPlayersChooseTrialsPhase : broadcast match_data_ready
    WaitPlayersChooseTrialsPhase --> MatchLoop : all players chose or time expired

    WaitAllPlayersReadyPhase --> Aborted : at least one player disconnected
    WaitPlayersChooseTrialsPhase --> Aborted : at least one player disconnected

    state MatchLoop {
        [*] --> RoundStart : broadcast round_start
        RoundStart --> WaitPlayersChooseOneTrialPhase
        WaitPlayersChooseOneTrialPhase --> RegisterAllTrialsAndCalculateMatchModifier : all player chose a trial or timeout
        RegisterAllTrialsAndCalculateMatchModifier --> BroadcastTrialsAndMatchModifier
        BroadcastTrialsAndMatchModifier --> WaitPlayersToBeReadyForQuestionPhase
        WaitPlayersToBeReadyForQuestionPhase --> SendNextQuestion : all players sent the ready state or timeout
        SendNextQuestion --> WaitForPlayersToBeReadyToDisplayTheQuestion
        WaitForPlayersToBeReadyToDisplayTheQuestion --> WaitForAnswersPhase : all players send the ready state or timeout
        WaitForAnswersPhase --> CalculateScore : all players answered or timeout
        CalculateScore --> CalculateRoundResult
        CalculateRoundResult --> BroadcastRoundResult
        BroadcastRoundResult --> WaitAllPlayersSeeResultPhase
        WaitAllPlayersSeeResultPhase --> RoundStart : rounds remain AND (all players sent ready state OR timeout)
        WaitAllPlayersSeeResultPhase --> [*] : no more rounds AND (all players snet ready state OR timeout)
    }

    MatchLoop --> Aborted : at least one player disconnected
    MatchLoop --> MatchEnd : loop done
    MatchEnd --> [*]

    Aborted --> [*]
```

And here the FSM for the client-side for a match:

```mermaid
stateDiagram-v2
    [*] --> ReceiveMessageThatPlayersWereFound
    ReceiveMessageThatPlayersWereFound --> LoadingAllResourcesAndWaitingForOpponentsDataPhase
    LoadingAllResourcesAndWaitingForOpponentsDataPhase --> SendMessageReadyForMatch : all resources loaded
    SendMessageReadyForMatch --> PlayersShowcaseIntroScene
    PlayersShowcaseIntroScene --> ChooseMatchTrialsScene : animation is done
    ChooseMatchTrialsScene --> WaitingForEveryoneToBeReady : player has chosen the trials or timeout
    WaitingForEveryoneToBeReady --> MatchLoop : received message from server to start match

    state MatchLoop {
        [*] --> ChooseRoundTrailScene
        ChooseRoundTrailScene --> WaitForEveryoneToChooseATrial : player has chosen a trial or timeout
        WaitForEveryoneToChooseATrial --> RoundModifiersScreen : received message from server
        RoundModifiersScreen --> SendMessageReadyForQuestion : animation is done
        SendMessageReadyForQuestion --> WaitForQuestionDataFromServerPhase
        WaitForQuestionDataFromServerPhase --> PrepareQuestionScreen : received question data
        PrepareQuestionScreen --> SendMessageReadyToDisplayQuestion
        SendMessageReadyToDisplayQuestion --> WaitForConfirmationToDisplayQuestionPhase
        WaitForConfirmationToDisplayQuestionPhase --> QuestionScreen : received confirmation
        QuestionScreen --> SendAnswer : player answered the question
        SendAnswer --> WaitForOthersPhase

        QuestionScreen --> Timeout : player time expired
        Timeout --> WaitForOthersPhase

        WaitForOthersPhase --> RoundResultScreen : received round results
        RoundResultScreen --> SendMessageReadyForNextRound : rounds remain
        SendMessageReadyForNextRound --> WaitForOthersToBeReadyForNextRound
        WaitForOthersToBeReadyForNextRound --> ChooseRoundTrailScene : received confirmation

        RoundResultScreen --> [*] : no more rounds
    }

    MatchLoop --> MatchResultScreen
    MatchResultScreen --> [*]
```
